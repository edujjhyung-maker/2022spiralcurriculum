# 나선형 성취기준 지도 · 국어

국어(2022 개정) 성취기준을 드롭다운에서 고르면, 같은 영역 안에서 **1~2 · 3~4 · 5~6학년군**으로 이어지는 나선형 위계(계열)가 세로로 펼쳐지는 웹앱입니다.

## 파일 구성 (4개, 같은 폴더에 함께 업로드)
- `index.html` — 화면 구조(뼈대). `styles.css`·`data.js`·`app.js`를 불러옵니다.
- `styles.css` — 디자인(색·레이아웃). 보통 손댈 일 없음.
- `app.js` — 앱 엔진(화면·동작). 보통 손댈 일 없음.
- `data.js` — **국어 성취기준·계열·계통·참고연계 데이터. 내용은 여기서만 수정.**

## 표시 우선순위 (성취기준 선택 시)
1. **직접 위계(`chains`)** — 같은 영역에서 학년군으로 이어지는 나선형 계열.
2. **참고 연계(`strands`/`refs`)** — 직접 위계는 없지만 같은 계통이거나 개별 지정된 관련 성취기준을 점선·"참고 연계"로 표시(직접 위계 아님, 양방향).
3. **완전 단독** — 위가 모두 없으면 해당 성취기준만 표시.

## GitHub Pages에 올리기 (git 명령어 없이, 웹에서)
1. GitHub → 우측 상단 **`+` → New repository** → 이름 입력 → **Public** → **Create repository**
2. **Add file → Upload files** → `index.html`, `styles.css`, `app.js`, `data.js` 네 파일을 끌어다 놓고 **Commit changes**
3. **Settings → Pages → Build and deployment → Source: "Deploy from a branch", Branch: `main` / `/(root)`** → **Save**
4. 1분쯤 뒤 주소가 나옵니다: `https://<내아이디>.github.io/<저장소>/`

> 루트에서 바로 열리려면 파일명이 반드시 **`index.html`** 이어야 하고, 네 파일이 같은 폴더에 있어야 합니다.

## 데이터 편집 요약 (`data.js`)
- **성취기준 추가**: `standards`에 `"코드": { subject, area, band, text, kw }` 한 줄.
- **직접 위계 추가**: `chains`에 `spine`(학년군 순서)과 인접 노드 사이 `links` note.
- **계통(참고 연계)**: `strands`의 `"코드": "계통이름"` — 같은 계통끼리 참고 연계로 묶임.
- **개별 참고 연계**: `refs`의 `"코드": ["대상코드", ...]` — 계통/사슬로 표현 안 되는 개별 연결(자동 양방향).

## 참고
- 계열·참고 연계는 교육과정 문서에 명시되어 있지 않은, 내용 체계에 근거한 **교육적 해석**입니다.
- 수학·사회·과학·영어는 별도 앱으로 추가 예정입니다.

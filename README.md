# 나선형 성취기준 지도

성취기준을 드롭다운에서 고르면, 같은 영역 안에서 **1~2 · 3~4 · 5~6학년군**으로 이어지는 나선형 위계(계열)가 세로로 펼쳐지는 웹앱입니다.

## 파일 구성
- `index.html` — 화면 구조(뼈대). `styles.css`·`data.js`·`app.js`를 불러옵니다.
- `styles.css` — 디자인(색·레이아웃). 보통 손댈 일 없음.
- `app.js` — 앱 엔진(화면·동작). 보통 손댈 일 없음.
- `data.js` — **성취기준·계열·계통 데이터. 내용은 여기서만 수정.**
- `README.md` — 이 문서.

> 네 파일(`index.html`, `styles.css`, `app.js`, `data.js`)은 **반드시 같은 폴더**에 함께 두어야 합니다.

## GitHub Pages에 올리기 (git 명령어 없이, 웹에서)

1. GitHub 접속 → 우측 상단 **`+` → New repository**
2. Repository name 입력(예: `curriculum-map`) → **Public** 선택 → **Create repository**
3. 새 저장소 화면에서 **Add file → Upload files**
4. `index.html`, `styles.css`, `app.js`, `data.js`, `README.md` 파일을 모두 끌어다 놓고 → **Commit changes**
5. 상단 **Settings → 좌측 Pages** 이동
6. **Build and deployment → Source: “Deploy from a branch”**, **Branch: `main` / `/root`** → **Save**
7. 1분쯤 뒤 새로고침하면 주소가 나옵니다:
   `https://<내아이디>.github.io/curriculum-map/`

> 루트에서 바로 열리려면 파일명이 반드시 **`index.html`** 이어야 합니다.

수정 후 다시 올릴 때: 저장소에서 `data.js`를 열고 연필(Edit) 아이콘 → 내용 수정 → **Commit changes** 하면 잠시 뒤 사이트에 반영됩니다.

## data.js 편집법

### ① 성취기준 추가
`standards`에 코드를 key로 한 줄 추가합니다.
```js
"6국03-05": {
  subject: "국어",
  area: "쓰기",
  band: "5-6",                 // "1-2" | "3-4" | "5-6"
  text: "쓰기 과정을 점검하고 조정하며 글을 고쳐 쓴다.",
  kw: [["점검","red"], ["고쳐 쓴다","blue"]]   // red=핵심개념, blue=글종류·행동
}
```

### ② 계열(나선형 사슬) 추가
`chains`에 학년군 순서대로 `spine`을 적고, 인접한 두 코드 사이 설명을 `links`에 넣습니다.
```js
{
  id: "쓰기 태도",
  spine: ["2국03-02", "4국03-05", "6국03-05"],
  links: {
    "2국03-02>4국03-05": { note: "쓰기 <b>흥미</b> → <b>자신감</b>으로 태도가 심화된다." },
    "4국03-05>6국03-05": { note: "자신감을 바탕으로 스스로 <b>점검·조정</b>하는 조절 능력으로 확장된다." }
  }
}
```
- `standards`에 등록한 **모든 성취기준**이 드롭다운에 나옵니다. `chains`에 들어간 성취기준은 학년군 위계(계열)가 함께 펼쳐지고, 계열이 없는 성취기준은 **단독으로 표시되며 "위계 없음" 안내**가 나옵니다.
- 한 학년군에 두 칸이 필요하면 사슬(chain)을 하나 더 만들면 됩니다.
- `note` 안에서는 `<b>강조</b>` 정도의 간단한 태그를 쓸 수 있습니다.

### ③ 새 과목 켜기
`subjects`에서 해당 과목을 `ready: true`로 바꾸고, 그 과목의 성취기준·계열을 `standards`/`chains`에 채우면 탭이 활성화됩니다.

## 참고
- 국어·수학은 2022 개정 교육과정 원문(내용 체계 및 성취기준)으로 검수해 반영했습니다. 사회·과학·영어는 예정입니다.
- 계열(이전↔이후 연결)은 교육과정 문서에 명시되어 있지 않은, 내용 체계에 근거한 **교육적 해석**입니다.

### 표시 우선순위 (성취기준 선택 시)
1. **직접 위계(`chains`)** — 같은 영역에서 학년군으로 이어지는 나선형 계열이 있으면 이를 펼침.
2. **참고 연계(`strands`)** — 직접 위계는 없지만 같은 **계통(strand)** 에 속한 다른 학년군 성취기준이 있으면, 점선·"참고 연계" 표시와 함께 관련 성취기준을 보여줌(직접 위계 아님).
3. **완전 단독** — 위 둘 다 없으면 해당 성취기준만 단독으로 표시.

### `strands`(계통) 편집
`data.js`의 `strands`는 `"코드": "계통이름"` 형태로, 같은 계통끼리 참고 연계로 묶습니다. 예:
```js
strands: {
  "2수03-10": "길이", "4수03-15": "길이",           // 같은 '길이' 계통
  "4수03-17": "양과 측정(들이·무게·넓이·부피)"        // 측정 계통 참고 연계
}
```
계통을 바꾸면 참고 연계 묶음이 바뀝니다. 계통이 지정되지 않은 성취기준은 '완전 단독'으로 표시됩니다.
